# RAG Research

Personal research project exploring how to build a reliable pipeline that processes arbitrary files into a knowledge base or graph, paired with an LLM for deep technical Q&A — essentially a personal search engine over your own content.

## What This Repo Tracks

- Research notes, decisions, and learnings
- A structured learning plan with progress tracking
- Reading notes and summaries of papers/articles
- Experiment logs and design decisions

## Directory Structure

```
notes/       — research notes, ideas, design decisions
reading/     — summaries and notes on papers, articles, blog posts
memory/      — session logs, context, things to remember
progress/    — progress tracking and milestones
```

## Key Areas of Exploration

1. **Document ingestion & parsing** — handling PDFs, code, markdown, and other file types
2. **Chunking strategies** — naive, semantic, structural, contextual
3. **Embedding models** — open-source vs API, dimensionality trade-offs
4. **Vector stores & hybrid search** — keyword + semantic retrieval
5. **Reranking** — cross-encoder models for retrieval quality
6. **Knowledge graphs** — entity extraction, GraphRAG, multi-hop reasoning
7. **LLM integration** — prompt design, citation, agentic RAG
8. **Evaluation** — RAGAS, test sets, measuring retrieval and answer quality
